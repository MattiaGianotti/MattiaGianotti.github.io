The Magic Room

The Magic Room is an innovative technological solution that allows to transform any room into a “smart” space in which lights, immersive projections on walls and floors, music, sounds, aromas, and physical materials are digitally controllable, programmable, and interactive. The result is a “Magic" space where children are exposed to multisensory stimuli and can interact with physical materials and multimedia contents through movements, gestures, and manipulation of objects. Magika, its operating system, allows to perform playful and educational activities (individual or collective) that stimulate all senses and are engaging. Through a tablet, teachers can control and configure each activity and personalize the experience according to the specific needs of children.

The Magic Room relies on a complex Internet of Things driven solution to Augmented Reality, realized through a multilayered multimodular architecture that ensures maintenaibility, upgrades and evolution to the project.

Timeframe of the project: 2018 - present

HOW DOES IT WORK?
Smartify the space!

In the creation of a Magic Room, we start from any empty room avaialble with at least 3x3 meters of spae on the floor and a white wall.

Then, we hide as much as posible the different technological elements from children's sight on the ceiling, inside the walls and embedded in everyday objects modified ad-hoc to be used as controllers of the space. Most of our appliances are off-the-shelf devices we integrated with API calls on the local network in order to increase stability and manageability of the system. When it was not avaialbe a commercial product, we used custom Smart Object. The final result is a human-driven approach to Internet of Things (IoT), usign a centralized control approach to coordiante the diffeent stimuli in response to the user actions identified. For motion tracking cameras and wall touch identifaction usign LiDar technology, artificial models have been used to analyze the data and determine the user interaction.

The technology inside can vary form installation of installations, but the main compoents are:
  1. two projectors, one projecting on the front wall (A) and one projecting on the floor
  2. audio dolby sorround 5.1 (C)
  3. Smart Material (D), material samples identifiable by the system through Smart Objects (E)
  4. Bubble emitter (F)
  5. Aroma emitter (G)
  6. Smart light system controllabel in intensity and color, either portable (H) or fixed on the ceiling (I)
  7. motion tracking device (J), now powered by a custom solution usign Google MediaPipe
  8. control tablet (K)
  9. Text To Speech system (L)
  10. LiDar sensors (M)

HOW DID WE IMPLEMENTED THEM?

To standardize the development process, allow students and researchers to create new experiences in the shortest amount of time possible, we have created a set of suport tool.

The most important elements is the starting Magic Room UnityPackage.
The UnityPackage I developed is a startup system for anyone who needs to create a new activity for the Magic Room. Among its several functions it
  • Offer the primitives to act on the different appliances distriburted in the Magic Room
  • Mask the details of the connections to the system
  • Offers the syncronization primitives for a lean intgration with the global system
  • Offers an setups scene and a generative game configuration menu creation
  • Automatically create the configuration detials, control panel and default standard package for the lean integration in the global system
The package offers both the classical c# scripts for the development, and editor scripts to wrap up the different models, offering a clearer set of functionalities, documentations, testing capabilities (available only in the Magic Room) and masking the more complex functionalities of the system.
HOW THE EXPERIENCE FLOW?

Each game has been designed individually and offers unique feeling and environemnt. Thus, we had to design a meta-narrative to encapsulate all the different games. The interplay between the different activities and this meta-narrative must be carefully managed to offer a lean experience. Plus, all experience should be controlled by the caregiver.

About the remote control system:
Our system must be controlled by caregivers while stayng with the children "in the field", hus not being able to use the mouse and keyboard of the controlling PC. For this reason we have designed a configurable tablet interface hat allows any user to control the system. In the tablet view it is possibile to:
  • select one of the available activities
  • configure the activity to play
  • pack up the activities to play in the session
  • select the currently playing children among the available players
  • use some general purpose commands available in any activity
  • simulate the behaviour of the sensors of the system in case of sensors' errors
  • terminate the experience or enter an emergency mode in case the user is having difficulties
